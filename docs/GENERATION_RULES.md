# 화살표 생성 규칙 (Arrow Escape)

`web/index.html`의 퍼즐 생성 로직 문서.

생성 파이프라인: `newGame()` → `placeTerrain()`(지형, §2.5) →
후보 8개 × (`generateSnakes()` + `fillGaps()`) → 최적 후보 선택 → `reseed()` →
`fillGaps()` → `pruneUnusedTerrain()`(이용 안 되는 파이프·포털 제거, §2.5).

공용 검사 함수 — 같은 규칙이 생성·연장·파종 어디서든 한 곳에서 관리된다:

| 함수 | 역할 |
|---|---|
| `traceRay(start, dir, grid?, selfId?)` | **공용 광선 추적기** — 탈출 광선의 단일 규칙 (§2.5). 아래 함수들이 모두 이 위에 서 있다 |
| `checkExit(s, id, grid)` | traced 광선 위의 충돌 검사 (생성·플레이 공용, §8) |
| `countBends(cells)` | 경로의 굴곡(방향 전환) 수 |
| `outwardHeadBanned(h, dir)` | 바깥 절반 영역에서 바깥 향 머리 금지 판정 (작은 구역 면제 포함) |
| `onLaterRay(list, id, e)` | 칸 e 가 "나중에 놓인 화살표"의 traced 탈출 경로 위인지 (`raySet` 캐시 조회) |
| `selfRayBlocked(cells, dir, grid, id, extra)` | 자기 몸이 traced 탈출 경로를 막는지 (j ≥ d 규칙) |
| `isSolvable(list)` | 탐욕 시뮬레이션으로 전체 풀이 가능성 검증 (안전망) |

## 핵심 보장: 항상 풀 수 있는 퍼즐

**역방향 구성** 원칙을 사용한다. 화살표를 하나 놓을 때마다 "이미 놓인 화살표들만 있는
상태에서 탈출 가능"한지 검사(`checkExit`)하고, 통과한 배치만 채택한다.

- 화살표를 없애는 것은 남은 화살표에게 항상 유리하기만 하므로(단조성),
  **놓은 순서의 역순으로 클릭하면 반드시 전부 사라진다.**
- 어떤 순서로 풀어도 막다른 상태가 되지 않는다. 한 번 자유로워진 화살표는
  다른 화살표가 제거되어도 다시 막히지 않는다.
- 안전망으로 생성 직후 `isSolvable()`(탈출 가능한 화살표를 반복 제거하는
  탐욕 시뮬레이션)을 한 번 더 돌려 검증한다.

## 1. 레벨 설정 — `levelCfg(lv)`

| 항목 | 규칙 |
|---|---|
| 격자 크기 | `min(28, 10 + lv*2)` — 첫 판부터 12×12, 레벨마다 +2, 레벨 9부터 28×28 고정 |
| 판 모양 | 사각형 → 하트 → 나무 → 동그라미 → 다이아 → 곰돌이 → 집 (7종을 계속 순환) |
| 직진 선호도 `straightBias` | `max(0.15, 0.5 - lv*0.04)` — 레벨이 오를수록 더 구불구불 |
| 최대 길이 `maxLen` | `min(20, 12 + lv)` 칸 |

## 2. 판 모양과 구역 — `cellZone()` / `buildMask()`

각 칸의 정규화 좌표 (u,v)∈[0,1]² 를 수식에 넣어 두 가지를 결정한다.

- **mask**: 화살표를 놓을 수 있는 칸인지 (모양 실루엣 안인지)
- **zone**: 색 구역. 화살표는 **자기 구역 안에서만** 생성·이동하고 그 구역의
  팔레트 색을 입는다. 구역 경계 덕에 모양·그림이 살아남는다.

| 모양 | 구역 → 팔레트 |
|---|---|
| 사각형·동그라미·다이아 | 단일 구역, 6색 순환 |
| 하트 | 빨강·핑크 계열 |
| 나무 | 잎(초록 계열) + 기둥(갈색) |
| 곰돌이 | 얼굴·귀(갈색 계열) + 주둥이(베이지) |
| 집 | 지붕(빨강 계열) + 벽(황토 계열) |

탈출 경로(직선 이동)는 구역·mask의 제한을 받지 않는다 — 제한은 몸통 배치에만 적용된다.

## 2.5. 지형(터널) — `placeTerrain()` / `traceRay()`

판에 고정된 두 유형의 지형. **화살표 몸통은 지형 칸에 배치될 수 없고**(걷기·연장·파종
모두 제외), 탈출 광선만 아래 규칙으로 통과한다. 지형은 `newGame()`에서 화살표보다
먼저 배치되며 게임 중 변하지 않는다.

**포털 (순간이동 쌍)** — 색으로 짝지어진 2칸. 광선이 포털 칸에 닿으면 짝 포털에서
나와 **같은 방향으로 계속 직진**. 연달아 여러 번 통과 허용. 점프도 거리 1칸으로 센다.

**파이프 (모양 있는 관)** — 모양 `-`(3~5칸), `ㄱ`/`ㄴ`(팔 2+2), `ㄷ`(2+3+2), `+`(5칸),
무작위 회전. 열린 입구에 **정방향**으로 닿으면 진입 → 관 모양을 따라 이동 → 반대쪽
출구 방향으로 직진(`ㄱ`/`ㄴ` 90° 회전, `ㄷ` 방향 반전, `+` 축별 직진 관통).
**옆면·역방향 입구는 벽**(충돌). 셀별 통과 규칙은 배치 시 `enter[진입방향]`으로
사전 계산해 둔다.

**개수 스케일**: 포털 쌍·파이프 각각 레벨 1~4 → 1개, 5~8 → 2개, 9+ → 3개 (`terrainCount`).

**배치 제약**: 지형끼리 최소 1칸 간격. 포털은 추가로 — 판 크기의 1/3 이상 서로 떨어짐,
같은 행/열 일직선 쌍 금지(광선 순환 예방), 가장자리 1칸 이내 금지, 작은 구역(30칸
미만: 해·구름 등) 금지(그림 보호). 배치 실패는 그냥 건너뛴다(최선 노력).

**`traceRay(start, dir, grid?, selfId?)`** — 직선 광선을 쓰던 모든 코드가 이 함수
하나를 쓴다. 머리가 지나갈 칸 목록(`{r,c,jump?}`, 포털 점프 칸은 `jump` 표시)과
종료 사유를 돌려준다: `'exit'`(판 밖) / `'wall'`(파이프 옆면·역방향, 광선 순환) /
`'hit'`(grid 를 준 경우 다른 화살표 칸 — 생성 핫패스 조기 종료). 거리 d = 목록
인덱스+1 로, 자기충돌 규칙(j ≥ d)과 애니메이션 호길이가 같은 단위를 공유한다.
포털 순환은 (칸, 방향) 방문 집합으로 감지해 `'wall'` 처리한다.

**풀이 보장과의 상호작용**: 배치 시점 `checkExit`가 traced 광선을 쓰므로 채택된
화살표의 광선은 반드시 `'exit'`로 끝난다 — 지형은 움직이지 않으니 파이프 옆면에
영원히 막히는 화살표는 아예 생기지 않는다. 화살표 제거의 단조성도 그대로라 역방향
구성 보장이 유지된다. `rayEmpty` 계산에서 지형 칸은 "덮일 수 있는 빈칸"에서 제외한다.
`onLaterRay`는 행/열 산술 판정이 불가능해져, 배치 시점에 각 화살표의 traced 경로를
`raySet`(Set)으로 캐시해 소속 검사로 대체했다(머리 연장 시에만 갱신).

**이용 안 되는 지형 제거** — `pruneUnusedTerrain()` (최종 보드에 1회): 완성된
보드를 탐욕 풀이로 시뮬레이션하며 탈출 광선이 지나가는 지형 칸을 모은다. 어떤
화살표도 통과하지 않는 파이프·포털 쌍은 순수 장식이므로 치우고, 비워진 칸은
`fillGaps`로 메꾼다. **제거가 안전한 이유**: 풀 수 있는 판에서는 살아남는 화살표가
파이프 옆면(벽)에 영원히 막힐 수 없고(막히면 클리어 불가) 포털은 애초에 막지
않으므로, "통과 안 되는 지형"은 어떤 광선과도 무관 → 제거해도 광선이 하나도
바뀌지 않는다.

## 3. 배치 순서 — 바깥 고리 우선 + 안쪽 무작위

화살표는 놓인 순서의 **역순**으로 풀리므로, 배치 순서가 곧 난이도 구조다.

- **바깥 고리(중심 반경 62% 밖)를 먼저** 배치한다: 빈 판일 때 중앙을 관통하는
  긴 탈출 경로를 얻고, 이후 배치들이 그 경로를 덮어 **게임 후반에야 풀린다.**
  → 시작 시 클릭 가능한 화살표가 안쪽에 생기고, "테두리부터 회전 클릭" 공략이 막힌다.
- **안쪽(반경 62% 안)은 무작위 순서**: 안쪽까지 바깥 우선으로 하면 중심부가
  탈출로를 못 찾아 구멍이 나므로(밀도 붕괴 실측), 안쪽은 무작위로 채운다.
- 각 키에 소량의 무작위 노이즈를 더해 판마다 순서가 달라진다.

## 4. 화살표 하나 만들기 — 시작 칸마다 최대 30~60회 시도

시도 횟수 = 기본 30회, 후반 패스(pass≥2)는 45회.

각 시작 칸에서 아래를 반복하고, 후보 중 최고 점수 배치를 채택한다.

**(a) 랜덤 걷기** — `randomWalk()`
- 목표 길이: 최소 3칸, `3 + rand(0 ~ maxLen-2)` 칸. 시도 10회 이후에는 최대
  6칸의 짧은 화살표로 전환해 빈틈을 메꾼다.
- 이동 조건: 격자 안 + 같은 구역 + 빈칸 + 자기 경로 미방문 (자기 회피 걷기).
- 방향 선택: 직전 방향 직진을 확률 `straightBias`로 선호, 나머지는 무작위.
- **마지막 걸음은 70% 확률로 판 중앙 쪽 방향을 선택** → 탈출 경로가 중앙을
  가로지르는 머리가 많아진다.

**(b) 정방향/역방향 평가** — 경로의 양 끝 중 어느 쪽을 머리로 할지 둘 다 평가.
`checkExit` 통과(이미 놓인 화살표만 있는 상태에서 탈출 가능)가 전제 조건.

**(c) 금지 배치 (trivial)** — 아래에 해당하면 그 방향은 채택 불가:
- `rayEmpty === 0`: 탈출 경로에 "나중에 화살표가 놓일 수 있는 빈칸"이 하나도 없음
  → 영원히 막힐 수 없는 공짜 화살표라 퍼즐에 불필요.
- `exitDist <= 2`: 머리가 판 가장자리 2칸 이내에서 바깥을 향함.
- `outwardHeadBanned()`: 머리가 **바깥 절반 영역**(중심 반경 50% 밖)에 있으면서
  중심 반대(바깥) 방향을 향함 (방향·반경 벡터의 cos > 0.3).
  단, 해·구름·나무기둥처럼 **30칸 미만의 작은 구역은 제외** — 그림 유지를 위해
  허용하며, 몇 개 안 돼 공략에 영향 없음.
- `straightLong`: 굴곡(`countBends() === 0`) 없이 5칸 이상 뻗은 직선 화살표.
- 모든 시도가 금지 배치뿐이면 그 시작 칸은 비워둔다 (이후 꼬리 연장으로 메꿔짐).

**(d) 점수식** — 후보 중 최고점 선택:

```
score = cov × 100          # 지금 자유로운 화살표의 탈출 경로를 덮는 수(가로막기) — 최우선
      + centerCross × 30   # 탈출 경로가 중앙 원반(반지름 = 판 크기의 30%)을 지나는 칸 수
      + exitDist × 8       # 탈출 경로가 판 안쪽으로 향하는 길이
      + 경로 길이            # 긴 화살표 선호(밀도)
      + 굴곡 수 × 2          # 구불구불한 모양 선호
      - noCross(5000)      # 생성 초반(배치 수 < 전체 칸/14)인데 중앙 미통과면 감점
      - len2(600)          # 2칸 화살표 감점 (후반 패스 pass≥2 에서는 빈틈 메꾸기용으로 면제)
      - straight4(300)     # 굴곡 없는 4칸 직선 감점
      - trivial(100000)    # 금지 배치
```

조기 종료: `cov ≥ 4`인 후보를 찾으면 남은 시도는 생략.

## 5. 배치 확정 후 상태 갱신

- 새 화살표에 덮인(탈출 경로 위에 몸통이 놓인) 기존 자유 화살표들을
  자유 목록(`freeSet`)과 경로 지도(`rayMap`)에서 제거 — 이제 "막힌" 상태.
- 새 화살표의 탈출 경로 중 빈 mask 칸들을 `rayMap`에 등록하고 자유 목록에 추가
  — 이후 배치가 이 칸들을 덮으면 이 화살표도 막힌다.

## 6. 다중 패스 (최대 5회)

전체 시작 칸 순회를 새 배치가 없을 때까지 최대 5번 반복한다.
1차에서 남은 빈칸(주로 가장자리 화살표의 짧은 탈출 경로 위)을 뒤 패스에서 덮는다.
후반 패스(pass≥2)는 시도 횟수를 30→45로 늘리고 2칸 화살표 감점을 풀어
빈틈 메꾸기에 집중한다.

## 6.5. 빈칸 메꾸기: 꼬리/머리 연장 — `fillGaps()` (모든 보드 후보에 적용)

금지 규칙 때문에 새 화살표를 놓을 수 없는 빈칸은 기존 화살표를 늘려 메꾼다.

**꼬리 연장** (`safeExtend`) — 머리(탈출 방향)는 그대로라 금지 규칙에 안 걸림:
- `onLaterRay()`: 연장 칸이 "나중에 놓인(=먼저 탈출해야 하는) 화살표"의 탈출 직선
  위에 있으면 안 됨 — 그 화살표가 자기 차례에 나가야 하므로.
- `selfRayBlocked()`: 자기 탈출 경로의 자기충돌 재검사 — 꼬리가 늘면 몸통
  인덱스가 1씩 밀려 나선형 자기충돌이 새로 생길 수 있음.

**머리 방향 연장** (`safeHeadExtend`) — 화살표가 자기 탈출 통로로 한 칸 전진한
모양. 남은 빈칸의 대부분이 탈출 통로 위의 칸이라 밀도 기여가 크다:
- `onLaterRay()` 검사 동일.
- 금지 규칙 재검사: 새 머리 기준 `exitDist > 2`, `outwardHeadBanned()`,
  직선 5칸 이상 금지(`countBends()`).
- `selfRayBlocked()` 자기충돌 재검사.

공통: 같은 구역(zone) 안으로만, 길이 상한 `maxLen + 4`.
더 이상 연장할 곳이 없을 때까지 반복 후 `isSolvable()` 안전망.

## 6.6. 추가 파종 — `reseed()` (선택된 보드에 1회)

연장까지 끝난 보드의 남은 빈칸에 모든 규칙을 지키는 새 화살표(3~10칸)를 한 번 더
심는다. 새 화살표는 항상 마지막 순서로 추가되므로(= 가장 먼저 풀리면 됨) 풀이
보장이 유지된다. 파종 후 `fillGaps()`를 한 번 더 돌려 마무리한다.

## 7. 보드 후보 선택 — `newGame()`

보드를 여러 개 생성해 가장 좋은 것을 고른다.

- 생성 횟수: 8개 (최대 판 28×28 기준 생성 시간 0.1초 안쪽).
- 1차 필터: 화살표 수 ≥ `max(6, mask칸수/20)` 그리고 `isSolvable()` 통과.
- 각 후보에 `fillGaps()`(연장)까지 적용한 **최종 밀도**로 비교한다.
- 2차 선택(2단계):
  1. 채움 칸 수가 최대치의 95% 이상인 보드만 남기고 (밀도 우선)
  2. 그중 `obvious×3 + freeCount` 최소인 보드 선택
     - `obvious` = 자유이면서 머리가 가장자리 2칸 이내인 화살표 수
     - `freeCount` = 시작 시 바로 탈출 가능한 화살표 수

## 8. 탈출 판정 — `checkExit()` (생성·플레이 공용)

머리에서 `traceRay()`로 얻은 광선 위의 칸을 순서대로 검사한다.

- 다른 화살표 칸 → 막힘.
- **자기 몸통 칸**: 경로 인덱스 `j`의 칸은 이동 `t > j`에 비워지고 머리는
  `t = d`(traced 호 단위 거리)에 도착하므로, `j ≥ d`면 제때 비켜주지 못해
  막힘(나선형 자기 충돌).
- 광선이 `'wall'`(파이프 옆면·광선 순환)로 끝나면 막힘.
- 아무것도 없으면 탈출 가능, `exitDist`(판 밖까지 traced 거리)와 광선(`ray`) 반환.

## 테스트 훅 — `window.__game`

브라우저 콘솔·자동화 테스트용 개발 인터페이스 (게임 로직과 무관, 배포 시 제거 가능):

| 메서드 | 역할 |
|---|---|
| `snakes()` | 현재 화살표 배열 (cells, dir, color, state) |
| `occ()` | 점유 그리드 (occ[r][c] = 화살표 id 또는 -1) |
| `size()` | `{cols, rows}` |
| `level()` | 현재 레벨 |
| `setLevel(lv)` | 레벨 변경 + 새 퍼즐 생성 |
| `stats()` | `{moves, budget, score, totalScore, combo, stars, timeAttack, timeLeft, gameOver, bestScore, bestTaLevel}` |
| `setTimeAttack(on)` | 타임어택 모드 켜기/끄기 + 새 런 시작 |
| `setTimeLeft(s)` | 남은 시간(초) 강제 설정 (타임오버 테스트용) |
| `clickCell(r, c)` | 해당 칸 클릭 시뮬레이션 |
| `terrain()` / `portals()` / `pipes()` | 지형 그리드·포털 쌍·파이프 목록 |
| `traceRay(r, c, dr, dc)` | 해당 칸·방향의 광선 추적 결과 (터널 경유 확인용) |
| `canExit(id)` | 화살표 id 가 지금 탈출 가능한지 (자동 풀이 루프용) |

사용 예 (자동 풀이 검증): 모든 화살표에 대해 탈출 직선이 뚫린 것을 찾아
`clickCell()`로 클릭하는 루프를 돌리면 항상 전부 사라져야 한다.
