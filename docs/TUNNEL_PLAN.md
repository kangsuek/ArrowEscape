# 터널 추가 설계 계획 (Arrow Escape)

두 유형의 터널을 추가한다. 둘 다 판에 고정된 **지형(terrain)** 이며, 화살표 몸통은
지형 칸 위에 배치될 수 없고, 탈출 광선만 규칙에 따라 통과한다.

## 확정 규칙 (사용자 결정)

**1번형 — 포털 (순간이동 쌍)**
- 색으로 짝지어진 2칸 한 쌍. 레벨당 1~3쌍.
- 탈출 광선이 포털 칸에 닿으면 짝 포털에서 나와 **같은 방향으로 계속 직진**.
- 연달아 여러 번 통과 허용.
- 시각 표현: 같은 색 고리(구멍) 2개.
- 등장: **레벨 1부터 1쌍**, 레벨에 따라 최대 3쌍.

**2번형 — 파이프 (모양 있는 관)**
- 모양: `-`, `ㄱ`, `ㄴ`, `ㄷ`, `+` (무작위 회전 적용).
- 열린 입구에 **정방향**으로 닿으면 진입 → 관 모양을 따라 이동 → 반대쪽 출구의
  방향으로 직진. `ㄱ`/`ㄴ`은 90° 회전, `ㄷ`는 방향 반전, `+`는 축별 직진 관통.
- **옆면에 닿으면 막힘(충돌)** — 관이 벽 역할을 겸한다.
- 출구 너머가 막히면 관 속으로 들어갔다가 왔던 길로 되감겨 나온다.
- 등장: **레벨 1부터 1개**, 레벨에 따라 최대 3개.

레벨 스케일(초안, 튜닝 대상):

| | 레벨 1 | 레벨 5 | 레벨 9+ |
|---|---|---|---|
| 포털 쌍 | 1 | 2 | 3 |
| 파이프 | 1 | 2 | 3 |

## 1. 데이터 모델

```js
// 지형 전역 (newGame 에서 생성)
let terrain = [];        // terrain[r][c] = null | 지형 셀 정보
let portals = [];        // [{a:{r,c}, b:{r,c}, color}]
let pipes = [];          // [{kind:'-','ㄱ','ㄴ','ㄷ','+', cells:[...], ends:[{cell, outDir}], path:[...]}]
```

- `terrain[r][c]`(또는 Map)로 O(1) 칸 조회:
  - 포털 셀: `{type:'portal', pair:{r,c}}`
  - 파이프 셀: `{type:'pipe', pipe, /* 진입 방향별 통과 정보 */}`
- 파이프의 셀별 통과 규칙은 **미리 계산**해 둔다: "이 셀에 방향 d로 진입하면
  → 다음 셀·나가는 방향 / 막힘". `+`는 두 축이 독립 직진.

## 2. 공용 광선 추적기 — `traceRay(start, dir)`

직선 광선을 쓰는 모든 코드를 이 함수 하나로 통일한다. **이 함수가 곧 게임 규칙**이다.

- 반환: 머리가 지나갈 칸 목록 `[{r, c, dir}...]` + 종료 사유(`'exit'` 판 밖 | `'wall'` 지형 옆면).
- 빈 칸 → 직진. 포털 칸 → 짝 칸으로 점프 후 같은 방향 직진(점프 구간을 목록에
  표시해 렌더링에서 건너뜀). 파이프 입구(정방향) → 내부 경로 따라 출구로.
  파이프 옆면/역방향 → `'wall'` 종료.
- 화살표(occ) 충돌 판정은 traceRay 가 하지 않는다 — 호출자가 목록을 순회하며
  occ 를 검사한다(현재 구조 유지).
- **무한 루프 가드**: 포털 배치에 따라 광선이 순환할 수 있다
  (예: B에서 나온 광선이 다시 A로 입사). `(칸, 방향)` 방문 집합으로 감지하면
  `'wall'`(막힘) 처리. 생성 단계에서도 같은 행/열 일직선 쌍은 피한다.
- 거리 `d` = 목록 인덱스(호 단위). 자기충돌 규칙(j ≥ d)과 애니메이션 호길이가
  같은 단위를 공유한다 — 포털 점프도 1칸으로 센다.

## 3. 교체 지점 (직선 광선 → traceRay)

| 위치 | 변경 내용 |
|---|---|
| `checkExit()` | 광선 순회를 traceRay 목록 순회로. exitDist = 목록 길이 |
| 생성 후보 평가 (rayEmpty·centerCross) | traceRay 목록 기준. **지형 칸은 rayEmpty 에서 제외**(덮일 수 없는 칸) |
| 탈출 경로 등록 (`rayCellsOf`/`rayMap`) | traceRay 목록의 빈 mask 칸만 등록 |
| `onLaterRay()` | 행/열 산술 판정 불가 → **배치 시점에 각 화살표의 traced 경로를 Set 으로 캐시**하고 소속 검사로 대체 (지형·머리 고정이므로 경로 불변) |
| `selfRayBlocked()` | traceRay 기준 j ≥ d 재검사 |
| `reseed()` | 후보 평가와 동일 변경 |
| `obvious` 계산 | traced 탈출 거리 기준 |
| 런타임 `buildPts()`/`tEnd` | 연장 구간 = traceRay 목록. 파이프 굽이는 폴리라인에 자연 포함(기차 연출 공짜), 포털 점프 구간은 그리기에서 스킵(몸통 두 조각 스트로크) |

되감김(bump) 애니메이션은 같은 경로를 역재생하므로 변경 불필요 —
"관에 들어갔다 되돌아 나오는" 연출이 그대로 나온다.

## 4. 지형 생성 (newGame 에서 화살표보다 먼저)

1. `buildMask()` 후 `placeTerrain(cfg)` 실행.
2. 포털 쌍: 빈 mask 칸 2개, 서로 판 크기의 1/3 이상 떨어지게. 같은 행/열
   일직선 배치 금지(루프 방지). 가장자리 1칸 이내 금지(무의미한 포털 방지).
   작은 zone(해·구름 등) 위 금지(그림 보호).
3. 파이프: 모양 무작위 선택 + 회전, mask 안 빈 영역에 맞으면 배치.
   지형끼리 최소 1칸 간격. 크기: `-` 3~5칸, `ㄱ`/`ㄴ` 팔 2+2, `ㄷ` 2+3+2, `+` 5칸.
4. 화살표 생성·연장·재파종은 지형 칸을 배치 불가 칸으로 취급
   (randomWalk·fillGaps·reseed 의 빈칸 판정에 지형 검사 추가).

## 5. 기존 난이도 규칙과의 상호작용

- **풀이 보장 유지**: 지형은 움직이지 않고 화살표 제거의 단조성을 해치지
  않으므로, 배치 시점 검사(checkExit)가 traced 광선을 쓰면 역방향 구성 보장이
  그대로 성립한다. 연장/재파종 안전 검사(onLaterRay·selfRayBlocked)도 traced
  기준이어야 보장에 구멍이 없다.
- 금지 규칙 재해석: `rayEmpty === 0`(영원히 못 막는 화살표)·`exitDist ≤ 2`·
  `outwardHeadBanned`(머리의 즉시 방향 기준 유지)·직선 5칸+ — 모두 traced
  광선/기존 정의 그대로 적용.
- centerCross 보너스도 traced 경로 기준 → 터널 경유 중앙 통과가 늘어나는
  부수 효과(난이도에 유리).

## 6. 렌더링

- 그리기 순서: 판 실루엣 → **파이프 몸통**(어두운 슬레이트, 굵기 cell×0.85) →
  화살표들 → **파이프 윤곽선**(테두리만 다시 그려 "안을 지나가는" 느낌) →
  포털 고리 → 파티클/텍스트.
- 포털: 이중 고리(밝은 테두리 + 어두운 구멍), 쌍 팔레트는 화살표와 겹치지 않는
  색(금·청록·자홍). 같은 쌍 = 같은 색.
- 파이프 입구: 끝을 살짝 벌린 개구부 표시.

## 7. 구현 단계 (각 단계마다 검증)

1. **terrain 모델 + traceRay 도입, 지형 0개로 기존 8곳 교체**
   → 지형이 없을 때 완전히 동일하게 동작하는지 회귀 검증(자동 풀이·지표·성능).
2. **포털 구현** (생성·렌더·애니) → 검증.
3. **파이프 구현** (생성·렌더) → 검증.
4. 레벨 스케일 적용·튜닝, GENERATION_RULES.md 에 터널 섹션 추가.

## 8. 검증 계획

- 자동 풀이: 레벨 1·5·12 × 각 5판 전부 클리어. 그중 "터널 경유 탈출"이 실제
  발생한 횟수를 카운트해 규칙이 작동함을 확인.
- 무한 루프 가드: 포털 일직선 강제 배치 시나리오에서 멈추지 않는지.
- 성능: traceRay 가 생성 핫패스이므로 50×50 생성 시간 재측정(예산 2~3초 유지).
- 기존 지표 유지 확인: 바깥 향 머리 0, 직선 5칸+ 0, 채움 밀도.
- 시각 확인: 포털/파이프 렌더, 파이프 굽이 통과 애니, 터널 되감김 충돌, 포털
  두 조각 몸통 그리기.

## 리스크

- `onLaterRay` 캐시 누락 시 풀이 보장 구멍 — 단계 1 회귀 검증으로 방지.
- 포털 광선 순환 — traceRay 가드 + 배치 제약 이중 방어.
- 가독성 — 지형 수 상한(각 3개)과 뚜렷한 색 구분으로 관리.
